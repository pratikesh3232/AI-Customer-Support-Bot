# AI Customer Support Chatbot

An AI-powered customer support chatbot built with LangChain and RAG. It answers questions grounded in your own knowledge base (FAQs, return policies, product manuals) and remembers the full conversation history across turns.

---

## Demo

> User: "What is your return policy?"
> Bot: "You can return items within 30 days of delivery. The item must be unused and in original packaging... [return_policy.txt]"

> User: "What if the item is damaged?"
> Bot: "If your product arrives damaged, contact us within 48 hours with photos... [return_policy.txt]"

The bot remembers previous messages ‚Äî so follow-up questions work naturally.

---


## Features

- **RAG-powered answers** ‚Äî retrieves relevant chunks from your knowledge base before answering
- **Conversation memory** ‚Äî remembers the full chat history so follow-up questions work naturally
- **Multi-document KB** ‚Äî supports FAQ `.txt`, return policy `.txt`, and product manual `.pdf`
- **Source citations** ‚Äî every answer references which document it came from
- **Streamlit chat UI** ‚Äî native `st.chat_message` bubbles with `st.chat_input`
- **One-time KB build** ‚Äî embed your docs once, reuse the FAISS index every session
- **Cached retriever** ‚Äî FAISS index loads once per session with `@st.cache_resource`

## Tech Stack

| Layer | Technology |
|---|---|
| LLM | GPT-4o-mini (OpenAI) |
| Embeddings | OpenAI `text-embedding-ada-002` |
| Vector store | FAISS |
| Memory | LangChain `ConversationBufferMemory` |
| RAG chain | LangChain LCEL |
| Document loaders | `TextLoader`, `PyPDFLoader` |
| Chunking | `RecursiveCharacterTextSplitter` |
| UI | Streamlit |

## Knowledge Base

The chatbot's knowledge comes from three documents in the `docs/` folder:

| File | Contents |
|---|---|
| `faq.txt` | Q&As covering payments, shipping, cancellations, and support |
| `return_policy.txt` | Full 8-section return, refund, and exchange policy |
| `product_manual.pdf` | TechNova X200 wireless headphones ‚Äî specs, controls, pairing, troubleshooting |

To add your own documents, place them in `docs/` and update the `loaders` list in `build_kb.py`, then re-run `python build_kb.py`.
